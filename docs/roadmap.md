## Phase 1 🚧 데이터 완성 + 의미 검색 (진행 중)

목적: **검토 정확도의 신뢰 기반**을 단단히 만든다. 키워드 hit이 없는 입력에서도
의미 기반으로 관련 사건을 찾아낼 수 있어야 한다.

**작업**:
1. **데이터 19건 추가 큐레이션** — 부마항쟁, 김대중 서거, 노태우 서거, 한일 위안부 합의일, 광화문 촛불 시즌, 4·11 총선/대선, 등
2. **pgvector 활성화**
   - `seed.ts`에 임베딩 생성 추가 (Ollama `nomic-embed-text` 또는 외부 API)
   - `review-engine.ts`의 `fetchNearbyEvents`를 키워드 검색 + 벡터 유사도 검색 하이브리드로 확장
3. **캘린더 DB 연동**
   - `apps/web/app/calendar/page.tsx`의 `SAMPLE_EVENTS` 하드코딩 제거
   - Server Component로 변환 → `events` 테이블 fetch
4. **문서 드리프트 해소**
   - `data-schema.md`의 `vector(1536)` → `vector(768)` 정정
   - `reviews.grade` 컬럼을 스키마에 추가
   - 마이그레이션 ALTER 스크립트 작성

**DoD**:
- `data/events/*.json` 50건 (목표 100% 큐레이션)
- 임베딩이 채워진 사건 비율 100%
- 캘린더 페이지가 DB 출처
- 회귀 테스트 미작성 키워드("욱일기" 등) 입력 시 LLM이 의미 매칭으로 사건 발견

**예상 기간**: 1~2주

## Phase 2 🧪 검증 자동화

목적: 데이터 추가 시 회귀가 발생하지 않도록 안전망을 만든다.

**작업**:
1. **회귀 테스트셋 작성** (`apps/web/test/regression.test.ts` 신규)
   - 5·18 + "탱크 시리즈" + "책상에 탁!" → F등급, 5·18·박종철 매칭
   - 4·16 + "리브랜드 런칭" → F 또는 D등급, 세월호 매칭
   - 8·15 + "광복 한정" + "독립" → A 또는 B등급
   - 7·11 + "여름 신상" → B등급
   - 10·29 + "할로윈 파티" + "이태원" → F등급
2. `pnpm test:regression` 스크립트 구현 — root `package.json`에 자리만 있음
3. (선택) Vercel preview 빌드 단계에 통합

**DoD**: 위 5개 케이스 모두 기대 등급으로 응답.

**예상 기간**: 3~5일

## Phase 3 🗣️ 사용자 인터뷰 (PMF 1차 검증)

목적: 첫 5명의 진짜 사용자에게 묻는다. "쓸만한가?"

**작업**:
1. 인터뷰 대상 모집 — 1인 브랜드 / SNS 운영 대행사 PMM 5명
   - 채널: 링크드인, 마케팅 커뮤니티, 지인 추천
2. 1:1 인터뷰 (30분)
   - 본인 캠페인 1건 가지고 와서 직접 검토
   - 검토 결과를 보고 "맞는가? 부족한가? 신뢰가 가는가?"
   - "다시 사용하겠는가?" / "지인에게 추천하겠는가?"
3. 단점·실수 사례 수집

**DoD**:
- "쓸만하다" 응답 **≥ 3/5**
- "다시 사용하겠다" 응답 **≥ 3/5**
- 단점·실수 사례 ≥ 5건 수집 (다음 이터레이션 인풋)

**예상 기간**: 1~2주 (모집 포함)

## Phase 4 🎯 PMF 후 이터레이션 (방향성)

Phase 3 결과에 따라 갈라진다. 미리 못 박지 않는다.

**시나리오 A — "쓸만하다" 응답 ≥ 4/5**
- 결제 모델 결정 (월 구독 / 검토 건당 / B2B)
- 사용자 인증 도입 (RLS 재설계 동반)
- 대행사 AE 페르소나 확장 — M4 닫힘 베타 (3개 대행사)
- 영문 README / 글로벌 데이터 큐레이션 시작

**시나리오 B — "쓸만하다" 응답 < 3/5**
- 핵심 불만 분석 → 검토 정확도 / UX / 데이터 커버리지 중 어디인지 파악
- 1~2가지 핵심 개선만 골라 재실험
- 페르소나 / 가치 명제 재검토 가능성

**시나리오 C — 중간 (3/5)**
- 가장 강한 사용 시그널 1건 더블다운
- 가장 약한 부분 1건 폐기 또는 보류
